🌍Overview

The Flutter Social Platform App is a cross-platform mobile application designed to deliver a seamless and engaging social experience. Built with Flutter and Dart, it leverages a feature-first clean architecture, GetX for state management, and Firebase for robust backend services. This app supports real-time social interactions, customizable user profiles, and a modern UI/UX, making it a scalable foundation for social networking.


This README is your guide to understanding the project, setting it up, and contributing effectively. Whether you're a new developer joining the team or an experienced contributor, you'll find everything you need to dive in!

✨Features

🔐 Authentication

  • Secure login/signup with email, phone, and OTP verification

🎨 UI/UX

🔧 Additional Features

  • Push notifications for real-time updates

🏗️Architecture Overview

We use a feature-first clean architecture, which organizes the codebase like a modular city:


🏬 Features: Independent modules (e.g., auth, profile, home) act like self-contained neighborhoods, each with its own UI, logic, and services.

🏛️ Core: Shared utilities (API clients, models, themes) serve as the city's infrastructure.

🧱 Widgets: Reusable UI components are like building blocks used across features.

⚙️ Services: External APIs and backend connections power the app's functionality.

Why This Architecture?

  • Modular: Add new features without disrupting existing ones.
  • Scalable: Supports large teams and complex apps.
  • Maintainable: Clear separation of concerns reduces bugs and simplifies debugging.
  • Collaborative: Developers can work on isolated features simultaneously.

🚀Quick Start for New Developers

Welcome to the team! Follow these steps to set up the project and start contributing:

📋 Prerequisites

  • Flutter SDK: Version ^3.5.3 (Installation Guide)
  • Dart SDK: Included with Flutter
  • IDE: Android Studio or VS Code with Flutter/Dart plugins
  • Git: For version control
  • Firebase CLI: For backend setup (Setup Guide)

🛠️ Setup Instructions

1. Clone the Repository

git clone https://github.com/your-org/flutter-social-platform.git
cd flutter-social-platform

2. Install Dependencies

flutter pub get

3. Configure Environment

cp .env.example .env

Edit .env with your API keys and Firebase configuration:

API_BASE_URL=https://your-api-endpoint.com
API_KEY=your-api-key
FIREBASE_PROJECT_ID=your-project-id
FIREBASE_API_KEY=your-firebase-key
ENABLE_ANALYTICS=true
ENABLE_CRASHLYTICS=true

4. Set Up Firebase

firebase login
flutterfire configure

5. Run Code Generation

flutter packages pub run build_runner build --delete-conflicting-outputs

6. Launch the App

flutter run

💡 Tips for New Developers

  • Start with main.dart: Understand app initialization and entry point.
  • Explore features/: Each folder is a self-contained module (e.g., auth/, profile/).
  • Use core/utils/: Contains helper functions for validation, formatting, and more.
  • Check widgets/: Reusable UI components to speed up development.
  • Ask Questions: Reach out on Slack (#flutter-social-platform) or email (flutter@rootpointer.com).

📐Architecture Guidelines

To maintain a clean and consistent codebase, adhere to these principles:

Principle Description Why It Matters
Feature Isolation Each feature (e.g., auth, profile) is self-contained with its own views, controllers, and services. Prevents code tangling and simplifies debugging.
No Logic in UI Business logic belongs in controllers, not in view/ files. Ensures reusable and testable code.
Reusable Core Use core/ for shared utilities like API clients, models, and themes. Reduces duplication and ensures consistency.
Minimal Dependencies Avoid heavy third-party packages unless justified. Keeps the app lightweight and maintainable.
Consistent Naming Follow naming conventions (e.g., *_view.dart, *_controller.dart). Improves code readability and navigation.

🔄Why GetX?

We use GetX for state management, routing, and dependency injection because it's:

⚡ Lightweight

Minimal performance overhead.

🔄 Reactive

Automatically updates UI when data changes.

🎯 All-in-One

Simplifies state, routing, and dependency management in a single package.

🧪Testing Strategy

Testing ensures our app is robust and reliable. The test suite is organized as follows:

test/
├── unit/                       # Tests for business logic
├── widget/                     # Tests for UI components
├── integration/                # End-to-end user flow tests
└── fixtures/                   # Mock data for testing

Testing Goals

  • Unit Tests: Verify individual functions and classes (e.g., controllers, services).
  • Widget Tests: Ensure UI components render correctly in isolation.
  • Integration Tests: Simulate real user journeys (e.g., login → post → comment).

Run Tests:

flutter test

Best Practices:

  • Write tests for new features before implementation.
  • Use Mockito for mocking dependencies.
  • Aim for at least 85% test coverage.

🤝Contributing

We welcome contributions from the community! To contribute:

Steps to Contribute:

1. Fork the Repository:

git clone https://github.com/your-org/flutter-social-platform.git

2. Create a Feature Branch:

git checkout -b feature/your-feature

3. Commit Changes:

git commit -m "Add your feature"

git push origin feature/your-feature
